# 前端组件

前端组件覆盖 UI 系统、管理后台、组件库和应用框架。

## shadcn/ui

- GitHub: https://github.com/shadcn-ui/ui
- 官网: https://ui.shadcn.com
- 模块: 前端 / UI
- 技术栈: TypeScript, React, Tailwind CSS, Radix UI
- 许可证: MIT
- 适合: 构建高度可定制的 React 界面，并希望把组件源码复制进项目自己维护。
- 不适合: 团队希望使用传统安装式组件库，并依赖严格版本升级。
- 接入成本: 低
- 替代方案: Ant Design, MUI, Chakra UI
- 评分: 5/5
- 备注: 适合需要视觉自主权的现代 React 应用。

## Ant Design

- GitHub: https://github.com/ant-design/ant-design
- 官网: https://ant.design
- 模块: 前端 / UI / 管理后台
- 技术栈: TypeScript, React
- 许可证: MIT
- 适合: 数据密集型后台、内部工具、表单、表格和企业管理系统。
- 不适合: 需要强品牌差异化的消费者产品，否则会花很多精力覆盖默认样式。
- 接入成本: 低
- 替代方案: MUI, Arco Design, Semi Design
- 评分: 4/5
- 备注: 很适合后台系统和 CRUD 密集型产品。

## Next.js

- GitHub: https://github.com/vercel/next.js
- 官网: https://nextjs.org
- 模块: 前端 / 全栈 Web
- 技术栈: TypeScript, React, Node.js
- 许可证: MIT
- 适合: 需要路由、SSR、静态生成、API 路由和部署约定的 React 应用。
- 不适合: 只需要极简静态前端，或项目明确不使用 React。
- 接入成本: 中
- 替代方案: Remix, Nuxt, SvelteKit
- 评分: 5/5
- 备注: 应把它视为应用基础框架，而不是小型可插拔组件。
