# 国际化与本地化组件

国际化与本地化组件覆盖多语言文案、翻译资源、日期数字格式、复数规则和前端框架集成。

## i18next

- GitHub: https://github.com/i18next/i18next
- 官网: https://www.i18next.com
- 模块: 国际化 / 通用多语言框架
- 技术栈: JavaScript, TypeScript
- 许可证: MIT
- 适合: Web、Node.js、React、Vue、移动端等多端项目需要统一翻译资源、命名空间、语言探测和插件生态。
- 不适合: 只需要非常少量静态文案切换，或项目希望完全依赖框架内置国际化方案。
- 接入成本: 中
- 替代方案: FormatJS, vue-i18n, Lingui
- 评分: 4/5
- 备注: 生态成熟但配置项较多，建议从资源结构、fallback 语言和命名空间规范开始设计。

## FormatJS

- GitHub: https://github.com/formatjs/formatjs
- 官网: https://formatjs.io
- 模块: 国际化 / ICU 消息与格式化
- 技术栈: TypeScript, JavaScript, React
- 许可证: MIT
- 适合: 需要 ICU MessageFormat、日期时间、数字、复数、相对时间和 React 国际化能力的项目。
- 不适合: 团队不熟悉 ICU 消息语法，或只需要简单键值翻译。
- 接入成本: 中
- 替代方案: i18next, Lingui, Globalize
- 评分: 4/5
- 备注: 对复杂语言规则支持好，但要建立提取、翻译、校验和缺失文案检查流程。

## vue-i18n

- GitHub: https://github.com/intlify/vue-i18n
- 官网: https://vue-i18n.intlify.dev
- 模块: 国际化 / Vue 多语言
- 技术栈: TypeScript, Vue
- 许可证: MIT
- 适合: Vue 项目需要组件级翻译、组合式 API、路由语言切换和本地化格式化。
- 不适合: 非 Vue 项目，或希望一套国际化核心同时覆盖多个前端框架。
- 接入成本: 低
- 替代方案: i18next, FormatJS, Lingui
- 评分: 4/5
- 备注: Vue 集成自然，适合和 Nuxt、组件库、内容管理系统一起设计多语言资源。
