# 组件条目模板

把下面这段复制到合适的 `catalog/*.md` 文件中。

```md
## 组件名称

- GitHub: https://github.com/org/repo
- 官网: https://example.com
- 模块: 分类 / 子分类
- 技术栈: 语言、框架、存储、运行时
- 许可证: SPDX 或许可证名称
- 适合: 这个组件最适合的项目形态。
- 不适合: 这个组件会带来过高成本或风险的项目形态。
- 接入成本: 低 / 中 / 高
- 替代方案: 方案 A, 方案 B, 方案 C
- 评分: 1/5 到 5/5
- 备注: 简短的实际选型判断。
```

## Review 清单

- GitHub 地址公开且正确。
- 许可证足够清楚，可以记录。
- 组件解决明确的模块级问题。
- 条目说明了什么时候不该使用它。
- 接入成本反映架构和运维影响，而不只是安装难度。
- 在合理情况下至少列出两个替代方案。
